# Sample Work Packet

## Intent Lock

Improve a sample documentation page without changing project behavior.

## Approved Envelope

Allowed:

- edit one documentation page
- improve clarity
- keep the example short

## Forbidden Downgrades

Do not:

- add automation claims
- add code execution
- change the project purpose
- remove the human review gate

## Acceptance Criteria

- wording is clearer
- scope remains docs-only
- no private project details are introduced
- result file is written

## Human Gates

Final acceptance requires human review.

## Validation Required

Manual review of changed documentation.

## Output Required

Write a result file at:

```text
docs/workflow/reviews/sample-result.md
```

## Branch / PR Rules

Use one branch and one pull request if this were a real repository task.

## What Not To Touch

- code
- deployment configuration
- secrets
- unrelated docs

## Stop Conditions

Stop if the task requires code, deployment, private data, or a new product decision.
